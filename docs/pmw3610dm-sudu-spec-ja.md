### PMW3610DM-SUDU センサードライバー仕様書

本仕様書は、PixArt PMW3610DM-SUDU 低消費電力レーザーマウスセンサーを制御するためのマイクロコントローラ向けドライバー開発仕様です。

#### 1. 基本仕様
*   **動作電圧** : VDD 1.7V〜2.1V (標準1.8V)、VDDIO 1.7V〜3.3V。
*   **通信インターフェース** : 3ワイヤーSPI (SDIO, SCLK, NCS)。
*   **SPIクロック周波数** : 最大 2 MHz。
*   **最大解像度** : 3200 CPI (200 CPIステップ)。
*   **モーションデータ** : 12ビット長。**負方向への移動量も正しく処理するため、データは「2の補数 (2's complement number)」フォーマットとして扱ってください**。

#### 2. ピンとハードウェア接続要件
*   **NCS (Chip Select)** : アクティブロー。通信の開始と終了を制御します。
*   **SCLK (Serial Clock)** : マイコンから供給するクロック入力。
*   **SDIO (Serial Data I/O)** : 双方向データライン。
*   **NRESET** : ハードウェアリセットピン（アクティブロー）。
*   **MOTION** : モーション検知割り込みピン（アクティブロー）。モーションデータが発生するとLowになります。**割り込み解除のためには、モーションレジスタ（0x03, 0x04, 0x05）を読み取るか、MOTIONレジスタ（0x02）に任意の値を書き込んでデータをクリア（High状態へ復帰）する必要があります**。
*   **XYLASER** : レーザー出力制御ピン。地絡（GNDへのショート）などの障害を検知した場合、安全のためレーザー駆動電流を遮断します。

#### 3. SPI通信仕様と厳密なタイミング制約
マスター（マイコン）主導で行う2バイト構成（1バイト目: アドレス、2バイト目: データ）の通信です。書き込み時はアドレスのMSBを「1」に、読み取り時は「0」にします。

##### 3.1. SPIクロックの有効化/無効化（必須シーケンス）
低消費電力化のため、任意のレジスタ操作の前後に以下のクロック制御が必須です。
1. 通信開始前：レジスタ 0x41 に 0xBA を書き込み、SPIクロックをオンにする。
2. **300μs 待機する**。
3. 目的の書き込み/読み取りオペレーションを実行する。
4. 通信終了後：レジスタ 0x41 に 0xB5 を書き込み、SPIクロックをオフにする。

##### 3.2. コマンド間の必須タイミング（遅延処理）
通信エラーやコマンドの無視を防ぐため、ドライバー側で以下のウェイトを厳密に実装してください。
*   **tSWW (Write to Write)** : 連続する書き込みコマンド間は **30μs** 待機する。
*   **tSWR (Write to Read)** : 書き込み直後に読み取りコマンドを送信する場合は **20μs** 待機する。
*   **tSRW / tSRR (Read to Next Command)** : 読み取り完了から次の通信開始までに **250ns** 待機する。
*   **tSRAD (Read Address-Data delay)** : 読み取り時のアドレス送信後、データ受信を開始するまでに **4μs** 待機する。
*   **tBEXIT (Burst Exit)** : バースト読み取り終了後、NCSをHighにして最低 **250ns** 維持する。
*   **tNCS-SCLK (NCSからSCLK開始)** : NCSをLowにしてから最初のSCLK立ち上がりまでに **120ns** 待機する。
*   **tSCLK-NCS write (書き込みNCS終了待機)** : 書き込みコマンド送信後、最後のSCLK立ち下がりからNCSをHighにするまでに **10μs** 待機する。
*   **tSCLK-NCS read (読み取りNCS終了待機)** : 読み取り完了後、最後のSCLK立ち下がりからNCSをHighにするまでに **120ns** 待機する。

#### 4. 初期化シーケンスと診断（セルフテスト）
電源投入時、またはソフトウェアリセット後に実行するシーケンスです。

##### 4.1. Power-up手順
1. VDDとVDDIOを供給する（遅延は100μs以内）。
2. NCSをHighにしてからLowにし、SPIポートをリセットする。
3. 少なくとも1フレーム（150μs）待機する。
4. レジスタ 0x2d に 0x00 を書き込む。
5. 10ms待機後、再度 0x2d を読み取り、下位4ビットがすべて 1 であることを確認する。
6. MOTIONピンの状態に関わらず、レジスタ 0x02, 0x03, 0x04, 0x05 を1回ずつ読み取る。
7. レジスタ 0x11 に 0x0d を書き込む。
8. レジスタ 0x1b に 0x04 を書き込む。
9. レジスタ 0x1c に 0x04 を書き込む。
10. レジスタ 0x1d に 0x0f を書き込む。
*(※注意: 0x3A への 0x5A 書き込みによるソフトウェアリセットを実行した後は、モーションデータが有効になるまで **3.5ms (tMOT-RST)** 待機する必要があります)*。

##### 4.2. SPI通信確認とセルフテスト（推奨）
初期化時に以下の診断を行うことで、システムの堅牢性が向上します。
*   **SPI通信確認** : PROD_ID (0x00) を読み取り 0x3e であること、NOT_PROD_ID (0x3f) を読み取り 0xc1 であることを確認します。
*   **セルフテスト手順** :
    1. レジスタ 0x3a に 0x5a を書き込みリセットする。
    2. レジスタ 0x41 に 0xba を書き込み300μs待機（SPIクロック有効化）。
    3. レジスタ 0x32 に 0x10 を書き込む。
    4. レジスタ 0x10 (SELF_TEST) に 0x01 を書き込む。
    5. 64フレーム分待機する。
    6. CRC0 (0x0c) == 0x73、CRC1 (0x0d) == 0xc4、CRC2 (0x0e) == 0xc1、CRC3 (0x0f) == 0xea を確認する。
    7. 完了後、再度チップをリセット(0x3a に 0x5a)して通常動作へ戻す。

#### 5. モーションデータの読み取りとエラー処理
データはバースト読み取り機能を使用して一括取得することが推奨されます。

##### 5.1. モーションバーストモード
BURST_READ (0x12) を使用して、最大10バイトのデータを連続して取得します。
1. NCSをLowにし、0x12 を読み取るアドレスを送信する。
2. **tSRAD (4μs) 待機する。ただし、この待機はRunモードでのウェイクアップ時のみ必須であり、省電力（Rest）モード中に読み取る場合はデータが常に利用可能なため待機は不要です**。
3. 連続してクロックを入力し、以下の10バイトを順に取得する。
    *   Motion: モーション状態とエラーフラグ
    *   Delta_X_L: X軸下位8ビット（2の補数）
    *   Delta_Y_L: Y軸下位8ビット（2の補数）
    *   Delta_XY_H: X軸/Y軸の上位4ビットずつ（2の補数）
    *   Squal: 表面品質スコア
    *   Shutter_Hi: シャッター開時間上位1ビット
    *   Shutter_Lo: シャッター開時間下位8ビット
    *   Maximum_Pixel: フレーム内最大ピクセル値
    *   Average_Pixel: フレーム内平均ピクセル値
    *   Minimum_Pixel: フレーム内最小ピクセル値
4. NCSをHighにして tBEXIT (250ns) 維持し、バーストを終了する。
5. 残存するモーションをクリアするため、0x02 に任意の値を書き込む。

##### 5.2. MOTIONレジスタ (0x02) のエラーフラグ監視
バーストデータの1バイト目（または 0x02 の読み取り）に含まれるフラグを評価し、エラーハンドリングを行います。
*   **Bit 7 (MOT)** : 1 の場合、新しいモーションデータが発生しています。
*   **Bit 4 (OVF: オーバーフロー)** : 12ビットバッファが溢れ、絶対パスデータが失われた状態です。読み取りでクリアされます。
*   **Bit 2 (LSR_FAULT: レーザー障害)** : XYLASERピンがGNDまたはVDDにショートしています（安全のためレーザーはオフ）。システムで警告を処理してください。
*   **Bit 0 (RST_FLAG: リセットフラグ)** : 意図せずパワーアップリセットが発生したことを示します。必要に応じて再初期化が必要です。

#### 6. 解像度（CPI）とセンサーの向き設定
解像度と、基板実装時の向き（オリエンテーション）を補正する設定は、SPIページ1の RES_STEP レジスタで行います。
1. 0x41 に 0xBA を書き込みSPIクロック有効化。
2. 0x7F に 0xFF を書き込み、ページ1へ切り替える。
3. 0x85（Zephyr表記では 0x05）に以下の構成で値を書き込む。
    *   **Bit 7 (SWAPXY)** : 1 でX/Y軸データを入れ替え
    *   **Bit 6 (INV_X)** : 1 でX軸移動方向を反転
    *   **Bit 5 (INV_Y)** : 1 でY軸移動方向を反転
    *   **Bit 4-0 (RES)** : 解像度 (例: 0x01=200cpi, 0x06=1200cpi, 0x10=3200cpi)
4. **0x7F** に 0x00 を書き込み、ページ0へ戻す。
5. 0x41 に 0xB5 を書き込みSPIクロック無効化。
*(※表記差分: データシート記載では RES_STEP=0x85、SPI_PAGE1=0xFF のようにページ1側の絶対番地で表現される一方、Zephyr実装ではページセレクタ 0x7F を使い、ページ1内オフセットとして RES_STEP=0x05 を扱います。どちらも同じ操作を指しています。)*

#### 7. 電源管理と低消費電力（Rest）モード
##### 7.1. シャットダウンモードとWakeup
*   **移行** : 0x3B に 0xE7 を書き込む（消費電流: 3μA）。SPIクロックはオフにしてください。
*   **Wakeup (復帰)** : 0x3A に 0x96 を書き込みます。復帰後、モーションデータが有効になるまで **16〜23ms (tWAKEUP)** の待機が必要です。

##### 7.2. 自動省電力 (Rest) モード
モーションがない場合、消費電流を抑える3段階のモードへ自動移行します。
*   **Run** : 平均 0.60mA
*   **Rest 1** : 消費電流 36μA。RUN_DOWNSHIFT (0x1b)、REST1_RATE (0x1c) で設定。
*   **Rest 2** : 消費電流 16μA。REST1_DOWNSHIFT (0x1d)、REST2_RATE (0x1e) で設定。
*   **Rest 3** : 消費電流 7μA。REST2_DOWNSHIFT (0x1f)、REST3_RATE (0x20) で設定。
*(※ PERFORMANCE (0x11) で強制的に各モードへ移行させることも可能です)*。

#### 8. トラッキングカバレッジ拡張アルゴリズム (Smart Algorithm)
大理石などの広範な表面での読み取りを強化するため、マイコン側でバースト読み取りの直後に以下のシャッター値（Shutter_Hi, Shutter_Lo）判定を実行してください。
*   **Smart Enable** : Shutter_Hi == 0 かつ Shutter_Lo < 45 の場合。 (0x41 に 0xba 書き込み -> 0x32 に 0x00 を書き込み -> 0x41 に 0xb5 書き込み)
*   **Smart Disable** : Shutter_Hi == 0 かつ Shutter_Lo > 45 の場合。 (0x41 に 0xba 書き込み -> 0x32 に 0x80 を書き込み -> 0x41 に 0xb5 書き込み)
*(※頻繁なSPIアクセスを減らすため、状態が変化した場合のみ書き込みを行うソフトウェアフラグ（ SW_SMART_Flag ）の実装が推奨されます)*。

#### 9. 追加機能：画像取得機能（Frame Capture / Pixel Grab）
製造時のレンズ焦点（Z-Height）チェックや高度なキャリブレーションに利用できる画像データ取得機能が搭載されています。
*   **Frame Capture (0x36)** : FRAME_GRABレジスタに `0x80` を書き込み、BURST_READ (0x12) を使用することで、現在のフレームの全ピクセル値をストリーミング取得できます。
*   **Pixel Grab (0x35)** : PIXEL_GRABレジスタを使用し、センサーの捉えた22x22のピクセルアレイ（484ピクセル）を座標ごとに個別に読み出し、全体画像を構築することが可能です。
